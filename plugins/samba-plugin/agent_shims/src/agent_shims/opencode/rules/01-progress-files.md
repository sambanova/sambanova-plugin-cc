# Progress Files

Task got progress/tracking file? It hold what change, why, what done.

Update it often, not just at end. Write as you go. File also serve as scratchpad: dump intermediate thought, open question, key thing to remember, so nothing lost between steps.

Before you conclude task — exit OR finish — update progress file. On conclude it MUST match reality:

- Mark done steps done.
- Drop or fix stale plan that no longer true.
- Note what skip + what fail, not just what pass.

Never end task with progress file that lie about state. Last write = accurate snapshot of where task land.
